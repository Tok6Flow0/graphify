# Token Usage Tracking Notes

Use this reference when updating `scripts/token_efficiency.py`, changing the telemetry schema, or reconciling estimates with provider usage.

## What Is Exact

- OpenAI's Responses input-token count endpoint returns exact input tokens for a request-shaped payload before sending it to the model: `POST /v1/responses/input_tokens`.
- Provider response usage is the best source for completed work when available. The Responses API reports total output tokens, including tokens generated by the model that may not appear in visible text.
- Organization Usage and Costs APIs can group usage by model, project, API key, user, service tier, and cached input tokens. Use those APIs for account-level reconciliation, not for every lightweight local run.

## What Is Estimated

- Local tokenizers are good for plain text but do not fully account for images, files, tools, schemas, request framing, changing model tokenization, prompt caching, or hidden reasoning tokens.
- The repo script therefore stores an estimation method and confidence. It hashes any input/output text used for estimation and does not store the raw text.
- Hidden reasoning output is approximated from visible output and reasoning effort unless provider-reported output tokens are supplied.

## Merge Policy

- Write one immutable event file per run under `.codex/token-usage/events/YYYY/MM/DD/`.
- Event filenames include timestamp, branch slug, and event id so branches normally merge by adding files.
- Regenerate `summary.json` and `ledger.jsonl` from event files on `main` after branch merges:

```bash
python3 scripts/token_efficiency.py merge --write-summary
```

- If generated summary files conflict, keep either side and rerun the merge command. Do not hand-merge token totals.

## Visualization

- `python3 scripts/token_efficiency.py visualize` creates charts by day, branch, model, and tag.
- Matplotlib is optional. If it is not installed, the script writes SVG and HTML fallback charts.
- The same command also generates a Remotion package under `.codex/token-usage/visuals/remotion/`.
- The Remotion package has animated sheets for overview, timeline, branch, model, task kind, and tag usage.
- To preview or export it:

```bash
cd .codex/token-usage/visuals/remotion
npm install
npm run studio
npm run still
npm run render
```

## Source-Backed Caveats

- Prompt caching works automatically on eligible repeated prefixes and can reduce latency and input-token cost. Keep stable instructions first and dynamic context later.
- Subagents are not automatically cheaper: each subagent performs its own model/tool work. Savings come from graph-scoping, using cheaper worker models, avoiding duplicated reads, and keeping the coordinator focused on synthesis.
- The script's local estimates are meant for repo-level trend tracking and routing decisions. Use exact provider usage or the Usage/Costs APIs for billing-grade reconciliation.
